# 📅 회의실 예약 시스템 구현 과제 가이드

## 1. 룸 및 예약 현황 조회 (ReservationStatusPage)
- [x] **회의실 목록 불러오기**: `GET /api/rooms` 호출 및 표시
- [x] **날짜별 예약 현황 타임라인**: `GET /api/reservations?date=YYYY-MM-DD` 연동
- [x] **타임라인 시간 범위**: `09:00 ~ 20:00` (30분 단위) 고정
- [x] **예약 상세 팝오버**: 타임라인 블록 클릭 시 시간, 인원, 장비 정보 표시
- [x] **내 예약 목록**: `GET /api/my-reservations` 연동 및 취소 기능 (`DELETE`)
- [x] **예약 취소 컨펌**: "정말 취소하시겠습니까?" 다이얼로그 노출 후 처리

## 2. 회의실 예약하기 (RoomBookingPage)
- [x] **필터 패널 구현**: 날짜, 시작/종료 시간, 인원, 장비, 선호 층 입력
- [x] **URL 동기화**: 필터 조건 변경 시 쿼리 파라미터(`?startTime=...`) 반영 및 공유 가능
- [x] **실시간 필터링 로직**:
    - [x] 인원 수용량 (`capacity >= attendees`)
    - [x] 필수 장비 포함 여부
    - [x] 시간 중복 체크 (기존 예약과 겹치지 않는 방만 표시)
- [x] **정렬 기준**: 층수 오름차순 → 이름순
- [x] **예약 생성**: `POST /api/reservations` 호출 및 성공 시 메인 이동

## 🚀 다른 사람들은 무엇을 더 했나요? (심화 포인트)
다른 제출자들의 PR에서 공통적으로 발견된 "가산점" 요소들입니다.

1.  **타임라인 직접 예약 (Quick Action)**:
    - 타임라인의 빈 공간을 클릭하면, 해당 시간과 회의실이 선택된 채로 예약 페이지로 바로 이동하는 UX 개선.
2.  **비즈니스 로직 분리 (Clean Code)**:
    - 페이지 컴포넌트 내부의 복잡한 필터링/예약 로직을 `useBooking`, `useTimeline` 등 커스텀 훅으로 추출.
3.  **애니메이션 및 완성도**:
    - 팝오버가 뜰 때 부드러운 애니메이션 처리.
    - 로딩 상태(`isLoading`)일 때 버튼 내부에 스피너(Spinner) UI 적용.
4.  **유효성 검사 강화**:
    - 종료 시간이 시작 시간보다 빠를 때 뿐만 아니라, 최소 예약 시간(예: 30분 이상) 등을 체크하는 로직 추가.
